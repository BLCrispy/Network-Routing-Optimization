Graph attribute formulas
Graph density (sparse-to-dense ratio)
ρ = 2·|E| / (|V|·(|V|−1)) → approaches 0 for large sparse road graphs

Avg node degree (connectivity)
k̄ = 2·|E| / |V| (undirected) ≈ 5.47–5.63 across all scales

Graph circuitry index (cycle richness)
γ = |E| / (3·(|V|−2)) → 0 = tree, 1 = fully triangulated

Intersection density
I_density = Intersection_Count / |V| ≈ 0.784 (stable ratio)
Runtime complexity formulas
Dijkstra (with binary heap)
T(V,E) = O((V + E)·log V) → empirical fit: T ≈ α·(V·log V) + β·E·log V

A* (with admissible heuristic h)
T(V,E) = O((V + E)·log V) — same worst-case, but expands far fewer nodes.
Empirical advantage: T_A* ≈ T_Dijkstra / k where k ≈ 2–30× (heuristic quality)

Bellman-Ford
T(V,E) = O(V·E) → empirical fit: T ≈ α·V·E
At 500K nodes × 1.4M edges: V·E ≈ 7×10¹¹ — explains 300–470s runtimes

ACO (Ant Colony Optimization)
T = O(n_ants · n_iter · |E|) — stochastic, parameter-sensitive
Fails on large graphs due to pheromone matrix dimensionality
Memory complexity formulas
All three exact algorithms
M(V,E) = O(V + E) → empirical fit: M ≈ 2.87 + c·V [GB]
Regression intercept (base process overhead) ≈ 2.87 GB
Slope ≈ 9.1×10⁻⁶ GB/node (≈ 9.1 KB per node including adjacency lists)

A* additional memory
M_A* = M_Dijkstra + O(|open set|) ≈ identical in practice

ACO
M_ACO = O(|E|) for pheromone matrix → prohibitive at large scale
Path cost quality formulas
Optimality gap vs Dijkstra (baseline)
Gap% = (Cost_alg − Cost_Dijkstra) / Cost_Dijkstra × 100

A*: Gap ≈ 0–13.6% (mean ≈ 5.1%) — admissible heuristic, not always optimal on all tie-break configurations
Bellman-Ford: Gap = 0% always (same relaxation, exact optimal)
ACO: Gap ≈ 188–1,683% (mean ≈ 735%) — highly suboptimal, stochastic

Efficiency ratio (path quality per second)
η = 1 / (Gap% + 1) / Runtime_s → A* dominates at scale
Graph attribute correlation with runtime

Dijkstra runtime vs edges: Pearson r ≈ 0.89 (strong). Node count and edge count are co-linear (r ≈ 0.9999), so either attribute predicts runtime equally well. Avg degree and circuitry are nearly constant across scale — they do not predict runtime variation independently.
